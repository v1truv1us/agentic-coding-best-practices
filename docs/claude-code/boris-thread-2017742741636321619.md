## Full thread breakdown (sub-thread captured)
Using ThreadReader for this specific thread, these 10 practices were extracted:

1. **Do more in parallel**
   - Run 3–5 worktrees/sessions in parallel for throughput.
   - Use named worktrees + shell aliases for fast switching.

2. **Start complex tasks in plan mode**
   - Invest heavily in planning; implementation quality improves.
   - Re-plan when things drift instead of brute-forcing forward.

3. **Continuously improve CLAUDE.md**
   - After corrections, update instructions/rules to reduce recurrence.
   - Keep guidance iterative and measurable.

4. **Create and version your own skills/commands**
   - If repeated more than daily, codify as reusable skill/command.

5. **Delegate bug-fixing workflows**
   - Feed logs/threads/failing CI signals to agents and let them run bounded fixes.

6. **Prompt for adversarial verification**
   - Ask Claude to challenge/prove changes before PR.

7. **Improve terminal/UX ergonomics**
   - Use statusline, tab labeling, and environment optimizations.

8. **Use subagents deliberately**
   - Offload focused tasks to keep main context clean.

9. **Use agentic coding for analytics/data tasks too**
   - Leverage CLI/MCP/APIs for metrics retrieval and analysis.

10. **Use learning-focused modes when appropriate**
   - Explanatory/learning styles for onboarding and domain transfer.

## Architect implications
- Parallelism requires stronger integration gates.
- Plan-first should be required for medium/high complexity tasks.
- CLAUDE.md/rules updates should be conditional but systematic (especially after repeated failures).
- Verification prompts can be standardized into reusable review workflows.
